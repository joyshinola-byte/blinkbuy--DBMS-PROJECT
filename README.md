 E-Commerce Order Management Database System
 
 Project Overview

The **E-Commerce Order Management Database System** is designed to manage the complete workflow of an online shopping platform. It provides a centralized database to store and manage customer details, product information, supplier records, orders, payments, shipments, and customer reviews. The system improves data accuracy, minimizes redundancy, and supports efficient business operations.


Objectives

- Analyze the business requirements of the system.
- Identify the core database entities.
- Define functional and non-functional requirements.
- Document the business requirements.
- Prepare the Software Requirement Specification (SRS).
- Provide a foundation for database design and implementation.


 Documents Included

This repository contains the following documents:

1. Requirement_Analysis_Report.pdf
This document explains the project requirements, business scenario, stakeholders, business processes, functional requirements, non-functional requirements, project scope, assumptions, and constraints.

 2. Business_Requirement_Document.pdf
This document describes the business objectives, business requirements, stakeholder roles, business rules, project scope, risks, assumptions, constraints, and expected business outcomes.

 3. SRS_Document.pdf
This document provides the complete Software Requirement Specification, including system features, functional requirements, non-functional requirements, use cases, data dictionary, system constraints, and future enhancements.
